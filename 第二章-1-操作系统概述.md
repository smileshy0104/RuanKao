## 1.1 什么是操作系统？

**官方定义**：操作系统（Operating System, OS）是计算机系统的**资源管理者**，属于**系统软件**，能够对计算机里面的软硬件资源实施管理。

**通俗理解**：操作系统就像一个"大管家"，负责协调和管理计算机中的所有资源（CPU、内存、硬盘、外设等），让各种程序能够正常运行。

```mermaid
graph TD
    A[计算机系统结构] --> B[应用软件层]
    A --> C[操作系统层 ⭐]
    A --> D[硬件层]

    B --> B1[办公软件、游戏<br/>浏览器等]
    C --> C1[Windows、Linux<br/>macOS等]
    D --> D1[CPU、内存、硬盘<br/>外设等]

    B1 -.调用.-> C1
    C1 -.管理.-> D1

    style C fill:#fcc,stroke:#333,stroke-width:3px
    style C1 fill:#fcc,stroke:#333,stroke-width:2px
```

### 特征1：并发性 ⭐⭐⭐（最重要）

**核心概念**：并发 ≠ 并行

```mermaid
graph LR
    A[并发 vs 并行] --> B[并行 Parallel]
    A --> C[并发 Concurrent ⭐]

    B --> B1[真正的同时运行]
    B1 --> B11[多个CPU核心<br/>每个核心同时运行<br/>不同进程]

    C --> C1[微观串行<br/>宏观并行]
    C1 --> C11[通过分时调度实现<br/>看起来同时运行]

    style A fill:#f9f,stroke:#333,stroke-width:3px
    style C fill:#fcc,stroke:#333,stroke-width:3px
    style C1 fill:#fdd,stroke:#333,stroke-width:2px
```

**并发性的实现原理：分时调度**

假设场景：
- CPU有8个核心
- 系统中有100个进程需要运行
- 问题：8个核心如何运行100个进程？

答案：**分时调度**

```mermaid
graph TD
    A[分时调度的工作原理] --> B[给每个进程分配<br/>一个时间片<br/>如100微秒]

    B --> C[进程A运行100微秒 → 暂停]
    C --> D[进程B运行100微秒 → 暂停]
    D --> E[进程C运行100微秒 → 暂停]
    E --> F[... 轮流执行]

    F --> G[微观层面 微秒级]
    G --> G1[串行的<br/>轮流执行]

    F --> H[宏观层面 人眼观察]
    H --> H1[并行的<br/>人类无法感知<br/>微秒级停顿]

    style A fill:#f9f,stroke:#333,stroke-width:3px
    style G1 fill:#fcf,stroke:#333,stroke-width:2px
    style H1 fill:#cfc,stroke:#333,stroke-width:2px
```

**实例理解**：
- 你同时打开了音乐播放器、浏览器、Word文档
- 实际上CPU在微秒级别快速切换这些程序
- 但你感觉它们在"同时"运行
- 这就是**并发性**

> **记忆口诀**：微观串行、宏观并行

---

### 特征2：共享性

**定义**：操作系统中的资源（软硬件资源）可以被多个进程或线程共享使用，而不是被某个进程独占。

```mermaid
graph LR
    A[共享性] --> B[多个进程可以<br/>共享使用资源]

    B --> C[示例1：文件共享]
    B --> D[示例2：打印机共享]
    B --> E[示例3：网络资源共享]

    C --> C1[多个程序可以<br/>同时访问同一个文件]
    D --> D1[多个程序可以<br/>共享使用打印机]
    E --> E1[多个程序可以<br/>共享网络带宽]

    style A fill:#f9f,stroke:#333,stroke-width:3px
```

---

### 特征3：虚拟性

**定义**：将物理上独占的设备转变为逻辑上共享的虚拟设备。

**经典案例：打印机的虚拟化**

```mermaid
graph TB
    A[打印机虚拟化] --> B[物理层面<br/>公司只有1台打印机]
    A --> C[逻辑层面<br/>每个员工电脑上<br/>都有一台打印机]

    B --> B1[物理上唯一]

    C --> C1[员工A的电脑：<br/>显示有1台打印机]
    C --> C2[员工B的电脑：<br/>显示有1台打印机]
    C --> C3[员工C的电脑：<br/>显示有1台打印机]
    C --> C4[... 每个人都有]

    C1 -.虚拟化.-> B1
    C2 -.虚拟化.-> B1
    C3 -.虚拟化.-> B1

    D[效果] --> D1[每个人都可以<br/>共享使用这台打印机]

    style A fill:#f9f,stroke:#333,stroke-width:3px
    style B1 fill:#fcc,stroke:#333,stroke-width:2px
    style D1 fill:#cfc,stroke:#333,stroke-width:2px
```

**实现技术**：SPOOLing（假脱机技术）

> **记忆技巧**：虚拟性 = 物理独占 → 逻辑共享

---

### 特征4：不确定性

**定义**：系统中的进程执行顺序和执行时间是不可预测的。

```mermaid
graph TD
    A[进程执行的不确定性] --> B[进程A正在运行...]
    B --> C[突然需要某个资源<br/>如文件]
    C --> D[资源被进程B占用中]
    D --> E[进程A进入阻塞状态<br/>等待...]
    E --> F[等待时间不确定<br/>取决于进程B何时释放]
    F --> G[进程A恢复运行]

    H[原因] --> H1[系统资源有限]
    H --> H2[进程可能需要等待资源]
    H --> H3[等待时间不确定]

    style A fill:#f9f,stroke:#333,stroke-width:3px
    style E fill:#fcc,stroke:#333,stroke-width:2px
```

> **形象比喻**：进程运行就像"走走停停"，什么时候停、停多久都不确定。

### 重点类型详解

**1. 分时操作系统** ⭐⭐

- **核心机制**：时间片轮转
- **实现功能**：并发性
- **现状**：现代操作系统普遍采用分时机制

**2. 实时操作系统** ⭐

- **应用领域**：军工、航天、工业控制、嵌入式
- **核心要求**：

  - ② 反馈必须**正确**
- **举例**：工业机器人控制系统、飞行控制系统

### 简答题

1. **请说明并发和并行的区别。**

<details>
<summary>点击查看答案</summary>

**并行（Parallel）**：
- 真正的同时运行
- 需要多个CPU核心
- 每个核心同时运行不同进程

**并发（Concurrent）**：
- 微观串行、宏观并行
- 通过分时调度实现
- 看起来像同时运行，实际是快速切换

**关键理解**：
- 并行是硬件层面的同时执行
- 并发是操作系统层面的时间片轮转

</details>

---

2. **请说明虚拟性的含义，并举例说明。**

<details>
<summary>点击查看答案</summary>

**虚拟性的定义**：
将物理上独占的设备转变为逻辑上共享的虚拟设备。

**经典案例：打印机虚拟化**

- **物理层面**：公司只有1台打印机
- **逻辑层面**：每个员工电脑上都显示有1台打印机
- **效果**：每个人都可以共享使用这台打印机

**实现技术**：SPOOLing（假脱机技术）

**核心**：物理独占 → 逻辑共享

</details>
